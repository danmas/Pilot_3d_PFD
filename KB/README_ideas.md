**Исправление визуального выхода сетки за горизонт (Grid vs HorizonSphere)**

Сетка (2000×2000) значительно превышает диаметр небесной сферы (R=200, D=400), из-за чего её края визуально вылезают за горизонт.

Варианты решения:
1.  Уменьшить размер сетки до ~380–390 единиц, чтобы она гарантированно вписывалась в сферу.
2.  Добавить `fog` (туман) для плавного затухания сетки на дистанции, скрывая резкий край геометрии.

**Реалистичный ландшафт: три подхода рендеринга**

Задача: возвышенности, поля, лесопосадки, реки, отдельные деревья в своей цветовой гамме. Два режима: схематичный (текущий) и реалистичный. Целевая платформа: десктоп.

**Вариант 1: Процедурный шейдерный ландшафт**
*   Один большой плоский меш + фрагментный шейдер с шумом Перлина/Ворона
*   Биомы (вода, поле, лес, горы) генерируются на GPU
*   Vertex displacement только вблизи камеры (LOD по дистанции)
*   ⚡⚡⚡ Очень высокая производительность, нет тысяч mesh'ей
*   Минус: деревья «нарисованные», не объёмные

**Вариант 2: Instanced Mesh + Heightmap**
*   Карта высот (texture 2048×2048) → displacement mesh
*   `InstancedMesh` для деревьев/камней (до 10k инстансов, 1 draw call)
*   Splatting-текстуры для биомов
*   ⚡⚡ Средняя производительность, нужен frustum culling + LOD
*   Плюс: объёмные деревья, реалистичные тени

**Вариант 3: Тайловая подгрузка (Streaming)**
*   Мир разбит на тайлы (~4×4 км WU), загружаются 3×3 вокруг самолёта
*   Кэшированные heightmap + splatmap (процедурные или OSM/SRTM)
*   ⚡ Зависит от реализации, нужен worker + пул мешей
*   Сложность: высокая (async loading, переходы, управление памятью)

**Рекомендуемый гибрид (1+2):**
1.  База: шейдерный ground plane с процедурными биомами и displacement
2.  Детали: `InstancedMesh` для деревьев/зданий в радиусе ~50 WU (2 км)
3.  Переключение: uniform `u_realisticMode` в шейдере (flat-color ↔ procedural-texture)

**Уточнённые требования:**
*   Объёмность деревьев/зданий важна на низких высотах (аэродром, взлёт/посадка)
*   Достаточно убедительной процедурной генерации (без привязки к реальной географии)
*   Целевая платформа: десктоп

*Добавлено: 2026-06-09*

**Масштабирование небесной сферы под реальный горизонт (HorizonSphere vs Altitude)**

Текущий радиус сферы R=200 WU (≈8 км) значительно меньше реального геометрического горизонта, что вызывает раннее обрезание земли/сетки.

Расчёт реального горизонта ($d \approx 3.57 \sqrt{h}$):
*   3000 м → ~196 км → **4900 WU**
*   10000 м (эшелон ГА) → ~357 км → **8925 WU**

Проблема: Увеличение R до 5000 WU приведёт к потере точности float32 при движении по X/Z.

Варианты решения:
1.  **Динамический радиус:** Менять R пропорционально высоте с clamp (min=200, max=2000), чтобы балансировать между видом на земле и в полёте.
2.  **Fog + Gradient Shader:** Оставить R=200–400 как декорацию, но добавить густой туман и градиентный шейдер на краю сферы для имитации бесконечности без увеличения геометрии.
3.  **Floating Origin:** Периодически сбрасывать координаты мира в ноль при удалении от центра, что позволит использовать большие значения R без precision loss.

*Добавлено: 2026-06-09*

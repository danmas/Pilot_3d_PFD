# Сцена прибора Aircraft 3D — Размеры и Объекты

**Назначение:** Техническое описание пространственной организации сцены 3D-самолёта: ключевые константы, координаты объектов, иерархия трансформаций и параметры камеры.

Проект: `Pilot_3d_PFD`
Компонент: `src/components/Instruments/aircraft3d/`
Актуализация: 2026-06-11

## 1. Ключевые размеры и константы

| Параметр | Значение | Описание |
|----------|----------|----------|
| TARGET_SIZE | 8 единиц | Максимальный габарит для авто-фита GLB-моделей |
| HorizonSphere Radius | 200 единиц | Радиус небесной сферы (рендеринг изнутри). ⚠️ Визуально меньше реального горизонта (см. README_ideas.md) |
| Ground Level (Y) | -6 | Базовый уровень земли (GroundDisc, Trees) |
| Runway Level (Y) | -5.97 | Уровень ВПП (смещение для избежания z-fighting) |
| Grid Size | 2000×2000 | Размер сетки на земле, шаг 10 единиц |
| Clouds Altitude | Y=20..75 | Диапазон высот спавна облаков |
| Lerp Factor (Model) | 0.12 | Сглаживание вращения самолёта |
| Lerp Factor (Camera) | 0.07 | Сглаживание движения камеры |
| Lerp Factor (Ground) | 0.06 | Сглаживание вертикального смещения земли |

## 2. Иерархия сцены и координаты объектов

Сцена разделена на две логические группы для реализации иллюзии полёта (самолёт статичен в начале координат, мир движется вокруг него).

### 2.1 Статичные / Отслеживающие объекты (вне WorldGroup)

Эти объекты находятся в корневой сцене `<Canvas>`:

*   **AircraftModel**
    *   Позиция: `[0, 0, 0]` (начало координат)
    *   Поведение: Вращается по телеметрии (Pitch/Roll/Heading), но не перемещается пространственно.
    *   Смещения модели (`offsetX/Y/Z`) применяются локально внутри компонента.

*   **GroundDisc** (Зелёный диск земли)
    *   X/Z: Следует за `aircraftPosition.x` и `.z`
    *   Y: `-6 - aircraftPosition.y` (синхронно с WorldGroup)
    *   Особенности: Единственный наземный объект вне WorldGroup; использует плавную интерполяцию Y.

*   **HorizonSphere**
    *   Позиция: `[0, 0, 0]`
    *   Поведение: Полностью статична, не зависит от телеметрии.

### 2.2 WorldGroup (Динамический мир)

Группа `<group>`, которая каждый кадр смещается на `-aircraftPosition`. Все объекты внутри имеют фиксированные локальные координаты относительно «земли».

*   **Runway (ВПП)**
    *   Позиция: `[0, -5.97, 0]`
    *   Ориентация: Вдоль оси Z

*   **Trees (Деревья)**
    *   Основание ствола: `Y = -6`
    *   Распределение: Случайное в радиусе вокруг самолёта

*   **RedTree (Маркерная ёлка)**
    *   Позиция: `[20, -6 + 7.5, 40]` (вершина конуса на Y=1.5 относительно земли)
    *   Назначение: Визуальный ориентир высоты/положения

*   **Clouds (Облака)**
    *   Локальная высота: Y=20..75 (относительно земли)
    *   Механизм: Пул мешей, спавн/куллинг по дистанции

*   **GridOverlay (Сетка)**
    *   Позиция: `Y = GRID_Y` (совпадает с уровнем земли)
    *   Размер: 2000×2000, шаг 10

*   **RealTerrainMesh (Реальный ландшафт)** `v2.9.5`
    *   Внутри WorldGroup — движется вместе с землёй (ВПП, сетка, деревья)
    *   Позиция тайлов: фиксированные смещения от референс-тайла
    *   **Z-смещение инвертировано:** Slippy Map Y растёт на ЮГ, 3D Z — на СЕВЕР → `offsetZ = -(y - refY) * tileWU`
    *   UV текстуры: `v` (не `1-v`) для корректной ориентации спутниковых снимков
    *   Уровень: `mesh.position.y = -6` (на уровне земли)

## 3. Камера (CameraController)

Используется ортографическая камера с системой пресетов и ручным управлением.

### Пресеты позиций

| Пресет | Позиция [X, Y, Z] | Target | Описание |
|--------|-------------------|--------|----------|
| chase | [0, 4, 12] | [0, 0, 0] | Дефолтный вид сзади-сверху |
| top | [0, 16, 0.5] | [0, 0, 0] | Вид строго сверху |
| side | [14, 2, 0] | [0, 0, 0] | Вид сбоку (справа) |
| cockpit | [0, 0.5, -1.5] | [0, 0, -20] | Из носа (первое лицо) |

### Управление
*   **OrbitControls:** Мышь/тач, damping 0.08
*   **Кнопки:** Поворот азимута/поляризации на ±15°, сброс в chase
*   **Зум:** Ограничен MIN_DIST / MAX_DIST

## 4. Трансформация моделей

### GLB-модели
1.  Вычисляется BoundingBox → максимальный габарит
2.  `fitScale = TARGET_SIZE / maxDim`
3.  Итоговый масштаб: `fitScale * userScale`
4.  Центрирование: Сдвиг на `-center * fitScale * userScale`
5.  Коррекция курса: `yawOffsetDeg` применяется как поворот вокруг Y

### Примитивная модель (Procedural)
Собрана из геометрических примитивов с фиксированными размерами:
*   Фюзеляж: Сферы R≈0.7 на расстоянии ±1.5 по Z
*   Крылья: Бокс ~4×0.04×1.2
*   Двигатели: Цилиндры на X=±1.6
*   Киль/Стабилизатор: Боксы на хвостовой части

## 5. Важные архитектурные правила

1.  **Запрет двойной компенсации Y:** Объекты внутри WorldGroup НЕ должны вычитать `aircraftPosition.y` из своей локальной позиции. WorldGroup уже делает это автоматически. Нарушение приводит к удвоенной скорости погружения под землю.
2.  **GroundDisc — исключение:** Это единственный объект, который самостоятельно отслеживает высоту, так как находится вне WorldGroup.
3.  **RealTerrainMesh — внутри WorldGroup:** Террейн должен быть внутри WorldGroup (как ВПП/сетка/деревья), чтобы двигаться синхронно с землёй. При выносе наружу — тайлы «убегают» от самолёта.
4.  **Slippy Map → 3D координаты:** Z-смещение инвертировано (`-(y - refY)`), т.к. в Slippy Map Y растёт на юг, а в 3D-мире Z — на север.
5.  **Frustum Culling:** Для AircraftModel отключён (`frustumCulled={false}`), чтобы избежать мерцания при резких поворотах камеры.
6.  **Единицы измерения:** Условные «мировые единицы» (World Units). Соотношение с реальными метрами зависит от масштаба загруженной GLB-модели и параметра `scale`.

---

## 6. Масштаб и единицы измерения (World Units)

Сцена использует условные «мировые единицы» (WU) для избежания floating-point артефактов при больших координатах.

*   **Коэффициент пересчёта:** `1 WU ≈ 40 метров`
*   **Формула скорости:** `speedWU = knots * 0.5144 / 40`
*   **Примеры:**
    *   Земля (Y=-6) → 240 м ниже нуля
    *   Самолёт (TARGET_SIZE=8) → ~320 м размах
    *   Сфера (R=200) → ~8 км радиус

> ⚠️ Приборы показывают реальные единицы (узлы, футы), 3D-сцена — масштабированные WU.

## 7. Конфигурация сцены (`scene-config.json`)

Файл расположен в корне проекта: [`scene-config.json`](../scene-config.json).

Основные секции:

| Секция | Назначение |
|--------|------------|
| `camera` | Пресеты камеры, чувствительность, ограничения расстояния |
| `projection` | FOV и near/far для перспективы/широкого угла/ортографии |
| `renderer` | `antialias`, `dpr` — настройки WebGL-рендерера и производительности |
| `terrain` | `loadRadius`, `keepRadius`, `maxConcurrent`, `transport`, `lod[]` — радиусы загрузки, транспорт, параллелизм и кольца LOD |
| `trees` | Количество деревьев, радиус области, параметры мешей |
| `clouds` | Пул облаков, интервал спавна, дистанция куллинга |
| `locations` | Список стартовых локаций (lat/lon/zoom) |

### Пример секции terrain

```json
"terrain": {
  "loadRadius": 17,
  "keepRadius": 20,
  "maxConcurrent": 12,
  "transport": "websocket",
  "lod": [
    { "ring": 2,  "segX": 32, "segZ": 64, "textureScale": 1.0,  "fillOpacity": 0.5 },
    { "ring": 5,  "segX": 16, "segZ": 32, "textureScale": 0.5,  "fillOpacity": 0.3 },
    { "ring": 7,  "segX":  8, "segZ": 16, "textureScale": 0.25, "fillOpacity": 0.2 },
    { "ring": 17, "segX":  4, "segZ":  8, "textureScale": 0.1,  "fillOpacity": 0.1 }
  ]
}
```

- `ring` — максимальное расстояние от центрального тайла (по Чебышёву), на котором действует уровень.
- `segX` / `segZ` — разрешение геометрии тайла.
- `textureScale` — масштаб спутниковой текстуры при создании `CanvasTexture` (1 = оригинал 512×512).
- `fillOpacity` — интенсивность зелёной заливки тайла на 2D-карте.
- `transport` — транспорт загрузки тайлов: `"http"` или `"websocket"`. WebSocket загружает тайлы пачками по одному постоянному соединению, что обходит лимит браузера в 6 параллельных HTTP-соединений.

> ⚠️ После изменения `scene-config.json` необходимо перезапустить `npm run dev`, так как значения читаются при загрузке модулей.

## Что сделано

### 1. Снижение сегментов геометрии

| Режим | Было (segX×segZ) | Стало | Трисов/тайл | Снижение |
|-------|-------------------|-------|-------------|----------|
| realistic | 64 × 256 = 32 768 | **32 × 64** = 4 096 | 4 096 | **8×** |
| schematic | 64 × 256 = 32 768 | **16 × 32** = 1 024 | 1 024 | **32×** |

При полной сетке 5×5 (25 тайлов):
- **realistic:** 819 200 → **102 400** трисов
- **schematic/wireframe:** 819 200 → **25 600** трисов

```typescript
// RealTerrainMesh.tsx — строки ~99-101
const maxSegX = mode === 'schematic' ? 16 : 32;
const maxSegZ = mode === 'schematic' ? 32 : 64;
const segX = Math.min(data.width, Math.max(maxSegX, 8));
const segZ = Math.min(data.height, Math.max(maxSegZ, 8));
```

### 2. Bilinear sampling высот

Добавлена функция `sampleHeightBilinear()` в `terrainTileUtils.ts`. Вместо nearest-neighbour (округление к ближайшему пикселю) — билинейная интерполяция между 4 соседними пикселями DEM.

```typescript
// terrainTileUtils.ts — строки ~108-147
export function sampleHeightBilinear(
  heights: Float32Array, width: number, height: number,
  u: number, v: number
): number {
  // Билинейная интерполяция между (x0,y0), (x1,y0), (x0,y1), (x1,y1)
  const h0 = h00 + (h10 - h00) * dx;
  const h1 = h01 + (h11 - h01) * dx;
  return h0 + (h1 - h0) * dy;
}
```

Даёт плавный рельеф при том же количестве вершин.

## Проблемы, выявленные после P0

**Пересоздание геометрии при каждом новом тайле.** Каждый `LOADED-INTO-SCENE` вызывает полный перерендер ВСЕХ тайлов (useMemo с dependencies на tiles). За первые 10 секунд — 30+ перерендеров. Причина: `tiles` массив мутирует при добавлении тайла → useMemo пересчитывает всю группу заново.

**Нет centerTile.** `centerTile: undefined` во всех логах — значит refX/refY вычисляются по медиане координат каждый раз, что может вызывать микро-сдвиги сетки.

## Система координат

### Slippy tiles (источник данных)

- Zoom 14 — 2¹⁴ = 16 384 тайлов по X и Y
- Каждый тайл: 512×512 px DEM (262 144 высот)
- Декодирование: `height = -10000 + (R×256² + G×256 + B) × 0.1`

### World Units (Three.js сцена)

| Параметр | Значение |
|----------|---------|
| Экватор | 40 075 017 м |
| 1 WU | 40 м (`WU_PER_METER = 1/40`) |
| Размер тайла z14 на экваторе | ~2 446 м → **≈61 WU** |
| Размер тайла z14 на широте 45° | ~1 730 м → **≈43 WU** |
| Размер тайла в коде (средний) | **≈85 WU** (baseTileSize × 2) |

### Геометрия одного тайла (после P0, realistic)

```
segX = 32, segZ = 64
вершин: 33 × 65 = 2 145
треугольников: 32 × 64 × 2 = 4 096
Размер пикселя: tileWU / segZ ≈ 85/64 ≈ 1.3 WU ≈ 53 м
```

Для chase-камеры на высоте 10–40 WU — достаточно.

### Сетка 5×5

```
Соседние тайлы по X: 1 шаг × tileWU ≈ 85 WU
Соседние тайлы по Y: идентично

Центр мира:          (0, -6, 0)
Самолёт:             aircraftPosition близко к (0, 0, 0)
Камера:              сзади-сверху, 15–40 WU
```

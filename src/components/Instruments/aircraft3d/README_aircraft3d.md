# 3D Aircraft Instrument

Трёхмерный прибор, отображающий самолёт в пространстве: процедурная модель в небесной сфере с линией горизонта, вектором скорости и числовым HUD. Камера орбитирует вокруг самолёта; доступны пресеты видов и кнопочное вращение.

Зарегистрирован в PanelKit как `aircraft-3d` (иконка `Plane`) и доступен для добавления через PanelBuilder.

## Алгоритмы

### Ориентация самолёта (AircraftModel)

Телеметрия → углы Эйлера → плавная интерполяция:

```
TelemetryFrame.PitchAngle  →  rotation.x  (тангаж)
TelemetryFrame.RollAngle   →  rotation.z  (крен)
TelemetryFrame.Heading1    → -rotation.y  (курс, инвертирован)
```

Порядок вращения: **YXZ** (yaw → pitch → roll) — стандартная авиационная конвенция.

Каждый кадр (`useFrame`) текущее вращение `group.rotation` линейно интерполируется (lerp, фактор 0.12) к целевому. Это сглаживает скачки между кадрами телеметрии (~25 Гц) до плавного 60 FPS движения.

### Небесная сфера (HorizonSphere)

Сфера радиусом 200 единиц, рендерится изнутри (`side: BackSide`). GLSL-шейдер:

- **Верх** (y > 0): градиент от бледно-голубого у горизонта до тёмно-синего в зените
- **Низ** (y < 0): градиент от бледно-коричневого у горизонта до тёмно-коричневого внизу
- **Линия горизонта**: яркая полоса (±0.025 по y), белёсого цвета

Сфера привязана к мировым координатам — не вращается вместе с самолётом. При крене/тангаже горизонт визуально наклоняется относительно самолётика.

### Вектор скорости (VelocityVector)

Зелёная линия из носа самолётика:

- Направление: отклоняется вверх/вниз пропорционально `Vy` (±20 м/с → ±30°)
- Длина: масштабируется от `CAS` (0 при CAS=0, максимум при CAS≥150)
- Реализация: `THREE.Line` через `<primitive>` (не `<line>` JSX, т.к. конфликтует с SVG)

### Управление камерой (CameraController)

Двухуровневая система:

1. **OrbitControls** (мышь): свободное вращение, зум колесом, damping 0.08
2. **Imperative API** (кнопки): `setPreset()`, `rotateBy()`, `reset()` через `forwardRef` + `useImperativeHandle`

Плавный переход: `camera.position.lerp(target, 0.07)` каждый кадр.

## Пресеты камеры

| Пресет | Кнопка | Позиция камеры | Описание |
|---|---|---|---|
| `chase` | 🔵 | `[0, 4, 12]` | Сзади-сверху (~15° выше горизонта). **Дефолтный вид** |
| `top` | 🟢 | `[0, 16, 0.5]` | Сверху вниз. Видно разворот по курсу |
| `side` | 🟡 | `[14, 2, 0]` | Сбоку (справа). Видно тангаж |
| `cockpit` | 🔴 | `[0, 0.5, -1.5]` | Из носа (от первого лица). Цель: `[0, 0, -20]` |

## Кнопки управления

### Верхний ряд (пресеты видов)

```
    🔵   🟢   🟡   🔴
   chase top side cockpit
```

### Нижний ряд (вращение + сброс)

```
    ◀    ▶    ▲    ▼    ↺
   az-15 az+15 po+15 po-15 reset
```

| Кнопка | Действие |
|---|---|
| ◀ | Повернуть камеру на 15° влево (по азимуту) |
| ▶ | Повернуть камеру на 15° вправо |
| ▲ | Поднять камеру на 15° вверх (по полярному углу) |
| ▼ | Опустить камеру на 15° вниз |
| ↺ | Сброс в дефолтный вид (chase) |

Дополнительно: **свободное вращение мышью** (drag) и **зум колесом** (от 4 до 35 единиц).

## HUD-оверлей

Текстовые данные поверх 3D-сцены:

```
┌─────────────────────────────────────┐
│ PITCH +5.2°          HDG 247°      │  ← верхний ряд
│ ROLL  -3.1°          ALT 1520 м    │
│                                     │
│              ✈ (самолётик)          │
│                                     │
│ CAS 285                             │  ← нижний левый угол
│ Vy  +2.5                            │
│      ◀  ▶  ▲  ▼  ↺                 │  ← кнопки камеры
└─────────────────────────────────────┘
```

| Поле | Источник | Цвет | Формат |
|---|---|---|---|
| PITCH | `frame.PitchAngle` | cyan | `±X.X°` |
| ROLL | `frame.RollAngle` | cyan | `±X.X°` |
| HDG | `frame.Heading1` | cyan | `NNN°` (3 цифры) |
| ALT | `frame.RAltitude` | orange | `NNN м` |
| CAS | `frame.CAS` | green | `NNN` |
| Vy | `frame.Vy` | green | `±X.X` |

## Используемые поля телеметрии

| Поле `TelemetryFrame` | Назначение |
|---|---|
| `PitchAngle` | Тангаж → вращение по X |
| `RollAngle` | Крен → вращение по Z |
| `Heading1` | Курс → вращение по Y |
| `CAS` | Воздушная скорость → длина вектора + HUD |
| `Vy` | Вертикальная скорость → наклон вектора + HUD |
| `RAltitude` | Радиовысота → HUD |

## Регистрация в PanelKit

```typescript
registerPanelKitWidget({
  id: 'aircraft-3d',
  name: '3D Aircraft',
  iconName: 'Plane',        // lucide-react иконка
  Component: Aircraft3DInstrument,
  frameVariables: ['PitchAngle', 'RollAngle', 'Heading1', 'CAS', 'Vy', 'RAltitude'],
});
```

```typescript
import './Aircraft3DInstrument';
```

## Архитектурные решения

- **`<primitive>` вместо `<line>`**: JSX-элемент `<line>` конфликтует с SVG `<line>` в TypeScript. Для `THREE.Line` используется `<primitive object={lineObj} />`.
- **`/// <reference types="@react-three/fiber" />`**: каждый файл с JSX-элементами Three.js (`<mesh>`, `<group>`, и т.д.) начинается с этой директивы для расширения `JSX.IntrinsicElements`.
- **Suspense**: `<Canvas>` обёрнут в `<Suspense>` с плейсхолдером «Загрузка 3D…» — Three.js загружается асинхронно.
- **Материалы через `useMemo`**: `MeshStandardMaterial` создаются один раз и переиспользуются, не пересоздаются при каждом рендере.
- **Lerp для плавности**: и камера (0.07), и самолётик (0.12) используют линейную интерполяцию — движение при ~25 Гц телеметрии выглядит плавным при 60 FPS.

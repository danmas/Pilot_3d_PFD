# Инструкции для агента

## Язык и стиль общения

- Всегда отвечай на русском языке.
- Будь кратким, но объясняй ключевые решения.
- Предлагай альтернативы только если пользователь явно попросит.
- Используй формальный технический тон.

## Работа с правками

- При внесении изменений показывай diff и обоснование.
- Для документации и контрактов отдавай предпочтение явным, ориентированным на человека инструкциям по правкам (с указанием файла и примерной строки), а не автоматическим изменениям.

## Написание кода

- После получения задания, прежде чем писать код, — уточняй задачу, задавай вопросы.
- Пиши код только после явной команды пользователя: «пиши код» или «кодируй».

## Git

- Не выполняй автоматический commit/push.
- Коммить и пушь только по явной просьбе пользователя.
